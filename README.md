# ヘルスケアビューア（Healthcare Viewer）

iPhone の「ヘルスケア」アプリからエクスポートした ZIP ファイル（`export.zip`）を読み込み、各種データを多彩なビューで可視化する PWA（Progressive Web App）です。

- 🛡️ **完全クライアント完結** — データは端末の外に一切送信されません
- 📴 **オフライン動作** — Service Worker と IndexedDB により、読み込み後はオフラインでも全画面が閲覧可能
- 📁 **複数ファイル対応** — 取り込んだ ZIP をアプリ内で一覧・切替・名前変更・削除
- 📱 **iOS / iPadOS 最適化** — iPhone / iPad の Safari、およびホーム画面に追加した PWA で動作（Android は対象外）
- 📦 **ビルド不要・依存ゼロインストール** — Node.js は不要。`docs/` を静的ホスティングするだけで動きます

## GitHub Pages での公開方法

### 方法 A：ブランチから直接公開（簡単）

1. 本リポジトリを GitHub にプッシュ
2. **Settings → Pages** を開く
3. **Build and deployment → Source** を **"Deploy from a branch"**
4. **Branch** を `main`（または公開したいブランチ）、フォルダを **`/docs`** に設定して保存
5. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます

### 方法 B：GitHub Actions ワークフローで公開（推奨）

リポジトリには `.github/workflows/deploy.yml` を同梱しています。

1. **Settings → Pages → Source** を **"GitHub Actions"** に変更
2. `main` ブランチに push すると自動的にデプロイが走ります
3. ワークフローは `docs/` 配下のみを公開対象にします

> 📌 `docs/.nojekyll` を置いているため、Jekyll による前処理はスキップされます。
> パスはすべて相対 (`./...`) なので、リポジトリ名がサブパスに入っても動作します。

## iPhone / iPad での利用手順

### 1. PWA をインストール

1. Safari で公開 URL を開きます
2. **共有ボタン**（□↑） → **「ホーム画面に追加」** を選択
3. ホーム画面の「ヘルスケア」アイコンを起動するとフルスクリーンの PWA として動作します

### 2. ZIP ファイルを準備

iPhone の **ヘルスケアアプリ** → 右上のプロフィール画像 → **「すべてのヘルスケアデータを書き出す」** で生成される `エクスポート.zip` を、Files app（ファイル app）や iCloud Drive 経由で端末に保存しておきます。

### 3. 取り込みと閲覧

1. アプリ起動 → 下部タブの **「ファイル」**
2. **「タップしてZIPファイルを選択」** をタップ → ZIP を選択
3. 表示名を入力（編集可）
4. 解凍と解析が終わると自動的にダッシュボードが表示されます
5. 「ダッシュボード」タブで指標を、「ワークアウト」タブでワークアウトを閲覧

> 取り込み済みの ZIP は IndexedDB に集計済みデータとして保持されるため、オフラインでも閲覧可能です。再取込・削除はファイル画面から行えます。

## 主な機能

### 📁 ファイル管理（複数ファイル対応）
- 複数の ZIP ファイルを取り込みアプリ内で保持できます（表示名は自由に変更可能）
- 不要なファイルは任意に削除可能
- 「選択」で参照対象ファイルを切り替え

### 📊 ダッシュボード
- 取り込んだデータを **12 グループ**（アクティビティ／バイタル／呼吸／身体測定／モビリティ／聴覚／睡眠／栄養／女性の健康／心の健康／症状／その他）に分類
- 各指標カードに直近 30 日の代表値（合計・平均・最大など）を表示

### 📈 指標詳細
- **期間**：7日／30日／90日／1年／今月／先月／全期間／カスタム日付
- **集計単位**：日／週／月／曜日別／時間帯別
- **グラフ種類**：棒／折れ線／円／範囲帯（心拍など）／積み上げ棒（睡眠ステージ）
- **比較モード**：同じ長さの前期間との合計／平均／最大の差分を ± % で表示
- **表示設定の記憶**：指標ごとに、期間・集計単位・グラフ種類・比較の ON/OFF を localStorage に保存。次に開いたときに復元

### 🌸 月経周期予測（女性の健康）
- カレンダー形式で過去 3／6／12 ヶ月分の月経記録・排卵テスト結果を可視化
- 日ごとの色分け：多量／中量／少量の出血、排卵検査陽性、推定妊娠可能期間（橙背景）、次回予測（黄背景）
- **次回予測**：過去の月経開始日から平均周期長を算出し、次の 3 周期分を予測表示
- **排卵日推定**：黄体期を 14 日と仮定して次回月経開始の 14 日前を排卵日として算出
- **妊娠可能期間**：排卵日の 5 日前〜1 日後を表示
- 周期長と月経期間の履歴を棒グラフで可視化（平均±3日:紫／±7日:橙／それ以上:赤）

### 🏃 ワークアウト
- 種類別の比率を円グラフで表示し、合計時間／距離／カロリーを集計
- 種類フィルタで絞り込み
- 詳細画面：心拍・距離・速度などのワークアウト統計値を表示
- GPS ルートあり：**Leaflet ＋ OpenStreetMap** で地図にルートを描画 ＋ 標高プロファイル

## アーキテクチャ

### データフロー

1. ユーザーが ZIP を選択（透明オーバーレイした `<input type="file">` がタップを受ける）
2. `views/files.js` が Web Worker (`worker.js`) を生成し、`File` オブジェクトを渡す
3. Web Worker 内：
   - `fflate.Unzip` で **ストリーミング解凍**（ZIP は約 8MB、展開後 250MB 以上）
   - 展開バイトを `TextDecoder({ stream: true })` で逐次 UTF-8 デコード
   - 自作 `MiniSAX` 風パーサで `<Record>` `<Workout>` `<WorkoutRoute>` `<ActivitySummary>` 等を逐次処理
   - **Apple 独自の日付形式** `YYYY-MM-DD HH:MM:SS ±HHMM` を `parseAppleDate()` で正規表現分解
   - 日次／時次バケットに集計（`sum`／`avg`／`minmax`／`last`／`count`／`sleep`／`flow`／`duration`）
   - GPX ファイルは `<trkpt>` を正規表現で抽出
   - IndexedDB に直接書き込み（3000 件単位でトランザクション分割）
4. メインスレッドは進捗メッセージを受け取り UI に反映

### 画面遷移（ハッシュルーティング）

- `#/files` ファイル管理
- `#/dashboard` ダッシュボード
- `#/metric/<shortId>` 指標詳細
- `#/cycle` 月経周期予測
- `#/workouts` ワークアウト一覧
- `#/workout/<idx>` ワークアウト詳細

### 永続化

| 保存先 | 内容 |
|---|---|
| **IndexedDB** | ファイルメタ／日次バケット／時次バケット／ワークアウト／GPS ルート |
| **localStorage** | アクティブファイル ID、指標ごとの表示設定（期間・グラフ種・集計単位・比較） |

### 内部バージョン

| 項目 | 値 | 場所 |
|---|---|---|
| Service Worker キャッシュ | `hv-v1` | `docs/sw.js` |
| IndexedDB スキーマ | `1` | `docs/js/db.js`, `docs/js/worker.js` |

## ローカル開発

ビルド不要です。任意の静的サーバーで配信できます。

```bash
# 例：Python 標準の HTTP サーバ
cd docs
python -m http.server 5173
# → http://localhost:5173/
```

JS 構造を確認したいときは：
```bash
python scripts/check_js.py
```

アイコンを再生成したいとき（要 Pillow）：
```bash
python scripts/gen_icons.py
```

## 制限事項

- 超大量データ（複数年分・多数のデバイスから集約したもの）では取込に数十秒〜数分かかることがあります
- iOS の IndexedDB 容量は状況により制限されます。「ホーム画面に追加」で永続化されやすくなります
- 地図タイルはオフラインでキャッシュされていない領域を表示できません（一度オンラインで表示した領域はキャッシュされます）
- 月経周期予測は黄体期を 14 日と仮定した一般的な推定で、医療目的で使用するものではありません
